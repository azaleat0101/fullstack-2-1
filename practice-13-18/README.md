# Заметки — практики 13–17

## 1. Описание проекта

Прогрессивное веб‑приложение «Заметки» для создания текстовых заметок и напоминаний.

Приложение поддерживает:

- офлайн‑работу (App Shell + кэширование через Service Worker);
- установку на устройство как PWA;
- push‑уведомления о новых задачах и напоминаниях;
- кнопку «Отложить напоминание» прямо в уведомлении.

Проект выполнен в рамках практических работ №13–17 по предмету «Фронтенд и бэкенд разработка (часть 2)».

## 2. Стек 

- **Frontend:** HTML, CSS, JavaScript, Socket.IO (клиент)
- **Backend:** Node.js, Express, Socket.IO (сервер), `web-push`
- **PWA:** Service Worker, Web App Manifest, Web Push API

## 3. Функциональность по практикам

## Практика №13
- Реализовано офлайн‑приложение заметок: данные сохраняются в localStorage, а статические ресурсы кэшируются через Service Worker.
- В sw.js настроены базовые обработчики жизненного цикла: при install в кэш добавляются HTML, стили, скрипты и иконки, при activate удаляются старые версии кэша.
- Обработчик fetch перехватывает запросы и отдаёт ресурсы из кэша при отсутствии сети, благодаря чему интерфейс открывается и остаётся работоспособным офлайн 

## Практика №14
- Добавлен Web App Manifest
- Манифест подключён в index.html через <link rel="manifest">, добавлены мета‑теги для интеграции с мобильными платформами, а Service Worker дополнен кэшированием иконок и файла манифеста.
- Приложение протестировано как PWA: его можно установить на устройство, после установки оно открывается в отдельном окне и выглядит как самостоятельное приложение.


## Практика №15
- Реализован App Shell: index.html + базовые стили и скрипты кэшируются через service worker.

- В sw.js настроены стратегии:
    - для статических ресурсов — Cache First;
    - для страниц content/* — Network First с fallback в кэш.

- Приложение продолжает работать и показывать сохранённые заметки без сети (кэш + localStorage).

## Практика №16 
- Подключён Socket.IO:
    - клиент: const socket = io('https://localhost:3001');
    - сервер: обработчики событий newTask и taskAdded.

- При добавлении обычной заметки:
    - фронтенд отправляет событие newTask на сервер;
    - сервер рассылает taskAdded всем подключённым клиентам;
    - на клиенте показывается toast‑уведомление «Новая задача: …».

## Практика №17
- На главной странице добавлен чекбокс «Добавить напоминание», который раскрывает поле даты/времени.

- Заметки в localStorage теперь хранятся как объекты с полями id, text, reminder (timestamp или null).

- Клиент при создании заметки с напоминанием отправляет через WebSocket событие newReminder с id, текстом и временем.

- Сервер сохраняет данные напоминания в Map, рассчитывает задержку и через setTimeout по наступлении времени отправляет push‑уведомление всем активным подпискам.

- В Service Worker обработчик push показывает уведомление и, если передан reminderId, добавляет action‑кнопку «Отложить на 10 секунд».

- При клике по action SW вызывает /snooze?reminderId=…, сервер отменяет старый таймер и ставит новый, после чего посылает повторное уведомление «Напоминание отложено».
